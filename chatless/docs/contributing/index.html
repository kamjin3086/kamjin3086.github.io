贡献指南

感谢你对 Chatless 的关注！欢迎通过以下方式参与：

提交 Issue

  • 使用清晰的复现场景、期望行为与实际行为。
  • 附带平台与版本信息、日志或截图。
  • 提交 Bug 报告或功能请求

提交 Pull Request

  1. Fork 仓库并创建分支：feat/*、fix/*、docs/*。
  2. 保持提交信息语义化，关联 Issue（如有）。
  3. 确保本地构建通过，并更新相关文档。
  4. 发起 PR 并补充变更说明与截图/视频。

代码风格

  • 前端遵循 TypeScript/React/Next.js 最佳实践。
  • Tauri/Rust 端遵循 Rustfmt/Clippy 规范。
  • 尽量保持小而聚焦的变更，便于审查。

安全与隐私

请勿提交任何敏感密钥；若发现安全问题，可通过Github Issues或邮箱与维护者联系。
